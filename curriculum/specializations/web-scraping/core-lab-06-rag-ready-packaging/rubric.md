# Rubric: Core Lab 6

| Criterion | Strong Evidence |
| --- | --- |
| Packaging scope | Only quality-passed records become retrieval chunks. |
| Citation metadata | Every chunk preserves source URL, collected timestamp, title, and quality status. |
| Chunking | Long record text is split predictably without losing citation context. |
| Refusal rules | Stale, missing-provenance, and financial-advice boundaries are explicit. |
| Manifest | Counts records, packaged records, chunks, citations, source URLs, and refusal rules. |
| Reflection | Explains why RAG packaging is a data-quality step, not only a vector database step. |
