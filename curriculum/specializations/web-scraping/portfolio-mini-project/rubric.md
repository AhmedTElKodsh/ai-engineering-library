# Rubric

| Category | Excellent | Passing | Needs Work |
| --- | --- | --- | --- |
| Source Approval | Uses only approved sources and explains collection boundaries, attribution, and freshness | Uses approved fixtures with basic rationale | Uses sources without approval or attribution |
| Provenance | Preserves source ID, URL, title, timestamp, and method through raw and clean records | Preserves most required source fields | Drops fields needed for review |
| Quality Report | Separates accepted, blocked, duplicate, stale, and missing-provenance records with clear counts | Reports accepted and rejected records | Only reports a total or hides failures |
| RAG Packaging | Produces citation-ready chunks, manifest counts, refusal rules, and reviewer notes | Produces basic chunks and citations | Produces uncited text blobs |
| Ethics Reflection | Names attribution, freshness, non-advice, privacy, and production failure limits | Names the main safety limits | Oversells the package as complete or live |

## Learning Evidence Add-On

| Evidence | Excellent | Passing | Needs Work |
| --- | --- | --- | --- |
| Learner Logic | Explains current capability, new composition capability, failure caught, FinAgent/practical transfer, and next advanced doorway | Explains the main project purpose and transfer | Treats the project as disconnected parsing tasks |
| Failure Literacy | Reproduces and explains a realistic source, provenance, stale-data, or duplicate failure | Notes a common failure after tests run | Only reports pass/fail without interpretation |
| Scope Discipline | Completes the fixture-first minimum path and labels live scraping as optional extension | Completes the core task with minor scope drift | Turns the project into an unbounded crawler |
| Portfolio Evidence | Leaves technical, failure, explanation, and transfer evidence | Leaves most evidence types | Leaves no reviewable evidence beyond code |
