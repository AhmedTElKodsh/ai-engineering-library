# Weekly Enhancement Block

Use this block in each learner-facing week or core lab when the local README
does not already make the curriculum logic explicit.

## Learning Logic

| Question | Answer |
| --- | --- |
| What can I do now? | Name the exact prior skill this lesson assumes. |
| What new capability am I adding? | Name the concrete behavior, judgment, or workflow this lesson adds. |
| What failure does this help me catch? | Name the malformed input, unsafe output, weak evidence path, or regression this lesson exposes. |
| How does this improve FinAgent or a practical AI system? | Connect the behavior to a safer assistant, data pipeline, retrieval flow, tool boundary, workflow, or production gate. |
| What should I be able to explain afterward? | Name the 2-4 sentence explanation the learner should be able to give. |

## Minimum Path

Complete the README trace, the TODOs in `workbench.py`, the relevant tests, and
the reflection/evidence prompt. This is the required path for Course 1.

## Optional Enrichment

Add one edge case, one extra test, or one small comparison. Do not expand the
lesson into an advanced topic unless the module guide explicitly marks it
optional.

## Advanced Doorway

This lesson prepares for a later advanced topic. Name that topic in 1-3
sentences, then return to the required workbench.

## Evidence Portfolio

Leave the lesson with:

- technical evidence: code, test, fixture, eval, trace, or log
- failure evidence: broken case, debug note, refusal, or limitation
- explanation evidence: 2-4 teammate-readable sentences
- transfer evidence: how this supports FinAgent or a later AI system
