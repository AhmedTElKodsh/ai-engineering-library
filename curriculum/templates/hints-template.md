# Hints

Use these one layer at a time. Do not read the next hint until you have tried the current one.

## Hint 1: Restate the Problem

What are the inputs, outputs, and invalid cases?

## Hint 1.5: Find the Failure

Before changing code, name the first failing test, log line, malformed input, or missing evidence record. The failure is the lesson map.

## Hint 2: Choose the Shape

What data structure makes the next step easiest?

## Hint 3: Handle the Smallest Case

Make the simplest test pass before handling all cases.

## Hint 4: Add the Edge Case

Now handle empty input, missing fields, invalid values, or inconsistent data.

## Hint 5: Review

Read the test names and assertion messages. They describe the expected behavior.

## Hint 6: Explain Like a Teammate

Write 2-4 sentences explaining what changed, why it works, what it does not prove, and where the pattern will reappear in FinAgent or a later AI system.
