# Reference Behavior: Module 6 Week 1 FinAgent Capstone Kickoff

Scaffold: `curriculum/main-track/06-capstone-projects/week-01-build/workbench.py`

## Intent

This lesson should scope a portfolio-ready FinAgent capstone before implementation expands: allowed behaviors, non-goals, data sources, eval cases, and evidence ledger.

## Intended Behavior

- Define capstone scope with safe educational boundaries.
- Build eval cases that cover supported, unsupported, citation, refusal, and stale-data behavior.
- Score eval cases repeatably.
- Build a portfolio evidence ledger with implementation, tests, evals, traces, demo notes, limitations, and reviewer checks.

## Reviewer Edge Cases

- Scope should explicitly reject investment advice.
- Eval cases should include failure modes, not only demos.
- Ledger should name evidence a reviewer can inspect.

## Do Not Accept

- Capstone scope without non-goals.
- Missing refusal or abstention evals.
- Portfolio evidence that is only narrative.
