# Reference Behavior: Module 6 Week 2 Capstone Polish

Scaffold: `curriculum/06-capstone-projects/week-02-polish/workbench.py`

## Intent

This lesson should turn the capstone into a reviewable portfolio artifact: demo script, release evidence, limitation note, and interview defense.

## Intended Behavior

- Build demo scripts that cover reviewer path and safety behavior.
- Summarize release evidence and detect blockers.
- Mark clean releases ready only when checks pass.
- Write limitation notes that name safety boundaries.
- Build interview defenses that explain core tradeoffs.

## Reviewer Edge Cases

- Blockers should prevent release-ready status.
- Limitation notes should mention no investment advice and citation limits.
- Interview defense should cover tests, evals, retrieval, safety, and production tradeoffs.

## Do Not Accept

- Demo scripts that only show the happy path.
- Release-ready status with unresolved blockers.
- Limitation notes that oversell the system.
