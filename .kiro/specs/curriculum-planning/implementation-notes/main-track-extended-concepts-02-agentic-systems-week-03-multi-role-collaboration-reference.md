# Reference Behavior: Module 4 Phase 7 Multi-Role Review Workflow

Scaffold: `curriculum/main-track/extended-concepts/02-agentic-systems/week-03-multi-role-collaboration/workbench.py`

## Intent

This lesson should teach role assignment, minimal handoffs, role-specific review, escalation decisions, and traceable collaboration.

## Intended Behavior

- Assign roles and include safety review for high-risk tasks.
- Create handoffs with minimal necessary context.
- Collect one review per assigned role.
- Escalate outcomes when critical reviews appear.
- Return collaboration decisions with traceable role ownership.

## Reviewer Edge Cases

- High-risk tasks should always include a safety role.
- Duplicate roles should not produce duplicate review confusion.
- Critical review findings should override clean minor findings.

## Do Not Accept

- Full-context handoffs by default.
- Reviews without role ownership.
- Ignoring critical findings.
